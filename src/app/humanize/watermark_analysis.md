# AI Watermark分析レポート：Gemini生成テキストの検出問題

## 発見された問題

**重要な発見**: テストに使用した「臓器インタラクトミクスの可視化」テキストがGeminiで生成されたものである場合、ExpertChat AIが86-87%のAI検出率を示すのは**AI watermark**が原因である可能性が高い。

## AI Watermarkとは

### Geminiのwatermark技術
```yaml
Googleの実装:
  - SynthID: テキスト生成時の確率分布操作
  - 統計的パターン: 単語選択の微細な偏向
  - 検出不可能: 人間には読み取れない
  - 検出可能: AI検出器には明確に識別可能
```

### Watermarkの仕組み
1. **生成時埋め込み**: トークン選択確率の微調整
2. **統計的信号**: 特定の語彙・構文パターン
3. **検出器対応**: 専用アルゴリズムで高精度検出
4. **除去困難**: 表面的な変更では無効化不可

## 我々のアルゴリズムへの影響

### 現在の検出状況
```
元テキスト（Gemini生成）: 86% AI検出
↓ v1.0処理後
結果: 87% AI検出（効果なし）
↓ v2.0処理後  
予測: 13-15% AI検出（大幅改善予測）
```

### Watermark vs 人間化攻撃

#### 我々の攻撃手法の効果
| 手法 | Watermark除去効果 | 表層検出回避効果 |
|------|------------------|-----------------|
| **Unicode同形異義文字** | 低 | 高 |
| **統計的特徴操作** | 中 | 高 |
| **構文変更** | 中 | 中 |
| **認知バイアス誘発** | 高 | 高 |
| **複合攻撃** | 高 | 最高 |

#### Watermark除去のメカニズム
```python
# Watermark除去の可能性
1. 統計的分布の攪乱:
   - 語彙多様性の人工的向上 → Geminiパターン破壊
   - n-gramパターンの変更 → 統計的信号の除去

2. 構文レベルの変更:
   - 文構造の大幅変更 → 生成パターンの無効化
   - 語順変更 → 確率分布の撹乱

3. 認知バイアス注入:
   - 人間的表現の追加 → AI性の希釈
   - 専門的権威性 → 人間らしさの演出
```
